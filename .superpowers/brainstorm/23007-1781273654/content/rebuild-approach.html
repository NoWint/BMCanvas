ModCanvas 重构策略

5 个致命问题需要解决，三种重构深度

1

精准手术：修复致命问题，保留可复用代码

保留 Rust 后端 + 类型系统 + mock 层，重写前端架构和组件。修复图谱边为空、重构为图谱中心架构、补全诊断/导出、添加 Framer Motion。

优势

  • 保留已验证的 Rust 后端
  • 保留类型系统和 mock 层
  • 工作量最小

风险

  • 旧组件的隐式耦合可能拖慢重构
  • 渐进修改容易遗漏不一致
2

前端重写：保留 Rust 后端，前端从零开始

保留 Rust 后端（修复导出空壳），删除所有前端代码，按新架构从零构建。新架构：图谱中心 + 浮动面板 + 信息卡片节点 + Framer Motion 动画。

优势

  • 干净架构，无遗留耦合
  • 一次到位，不需要反复修
  • Rust 后端已验证可复用

风险

  • 前端全部重写工作量大
  • 可能引入新的 bug
3

完全重写：前后端都从零开始

删除所有代码，按 PRD 重新设计数据模型、API、组件。最彻底但工作量最大。

优势

  • 最干净，无任何遗留
  • 可以重新设计数据模型

风险

  • 工作量最大
  • Rust 后端大部分可复用，重写浪费
  • 风险最高

推荐：方案 2 — 前端重写

Rust 后端的核心逻辑（项目 CRUD、Modrinth 搜索、SQLite）已经验证可用，只需修复导出空壳。 前端架构问题太深（图谱不是中心、边永远为空、无动画、交互模型错误），精准手术会不断碰到旧代码的隐式耦合。 前端重写可以一步到位实现新架构，同时复用已验证的后端。
